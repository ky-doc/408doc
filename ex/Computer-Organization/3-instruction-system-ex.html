                            例题 一个计算机系统采用323232位单字长指令，地址码为121212位，若定义了250250250条二地址指令，则还可以有()条单地址指令。

                            A.4KA.4KA.4K

                            B.8KB.8KB.8K

                            C.16KC.16KC.16K

                            D.24KD.24KD.24K

                            解：DDD。地址码为121212位，二地址指令的操作码长度为32−12−12=832-12-12=832−12−12=8位，已定义了250250250条二地址指令，28−250=62^8-250=628−250=6，所以上一层多出了六种地址给单地址指令，即可以设计出单地址指令6×212=24K6×2^{12}=24K6×212=24K条。

                            例题 某计算机按字节编址，指令字长固定且只有两种指令格式，其中三地址指令292929条、二地址指令107107107条，每个地址字段为666位，则指令字长至少应该是()。

                            A.24A.24A.24位

                            B.26B.26B.26位

                            C.28C.28C.28位

                            D.32D.32D.32位

                            解：AAA。三地址指令有292929条，所以其操作码至少为555位。以555位进行计算，它剩余32−29=332-29=332−29=3种操作码给二地址。而二地址另外多了666位给操作码，因此其数量最大达3×64=1923\times64=1923×64=192。所以指令字长最少为232323位，因为计算机按字节编址，需要是888的倍数，所以指令字长至少应该是242424位，选AAA。

                            例题 设指令字长固定为161616位，试设计一套指令系统满足：

                            1）有151515条三地址指令

                            2）有121212条二地址指令

                            3）有626262条一地址指令

                            4）有323232条零地址指令

                            解：若要151515条三地址码，而一共要从零到三位的地址，指令字长为161616位。

                            所以三地址指令的操作码长度为4位，这样一共能得到0000 A1 A2 A30000\,A_1\,A_2\,A_30000A1​A2​A3​到1110 A1 A2 A31110\,A_1\,A_2\,A_31110A1​A2​A3​一共151515条三地址指令。1111 A1 A2 A31111\,A_1\,A_2\,A_31111A1​A2​A3​用于拓展。

                            而对于二地址指令，一定是以111111111111开始的，由于只需要121212条二地址指令，拓展A1A_1A1​作为操作码时，只用用到000000000000到101110111011就可以了，即而二地址指令从1111 0000 A2 A31111\,0000\,A_2\,A_311110000A2​A3​到1111 1011 A2 A31111\,1011\,A_2\,A_311111011A2​A3​。剩下的110011001100、110111011101、111011101110、111111111111都是以111111开头，所以1111 11XX A2 A31111\,11XX\,A_2\,A_3111111XXA2​A3​用于拓展。

                            对于一地址指令，需要626262条，而222的八次方减一为636363，所以一地址指令的操作码至少还要拓展888位。最后的111111111111一定要留给下一位拓展，所以1111 11001111\,110011111100到1111 11101111\,111011111110的后面一个操作码A2A_2A2​范围是000000000000到111111111111全部用到，这样一共满足了3×16=483×16=483×16=48个，还需要141414个，所以1111 11111111\,111111111111只还要满足141414个就可以了，即1111 1111 00001111\,1111\,0000111111110000到1111 1111 11011111\,1111\,1101111111111101，1111 1111 11101111\,1111\,1110111111111110和1111 1111 11111111\,1111\,1111111111111111留给零地址指令拓展。

                            而零地址指令必然以1111 1111 1111111\,1111\,11111111111111开头，正好余下1+4=51+4=51+4=5位也就是323232条零地址指令可以满足条件。

                            例题 假设指令字长为161616位，操作数的地址码为666位，指令有零地址、一地址、二地址333种格式。

                            1）设操作码固定，若零地址指令有MMM种，一地址指令有NNN种，则二地址指令最多有几种？

                            2）采用扩展操作码技术，若零地址指令有MMM种，一地址指令有NNN种，二地址指令有多少种？最多有几种？

                            3）采用扩展操作码技术，若二地址指令有РРР条，零地址指令有QQQ条，则一地址指令最多有几种？

                            解：操作符拓展的思想都是一样的，本层操作符支持指令条数等于当前空余位的指数乘上上一层指令空余的位数。

                            1）对于操作码固定，对于所有指令都是一样的操作码。由于最多二地址，所以操作码为16−2×6=416-2\times6=416−2×6=4位，所以一共24=162^4=1624=16条，二地址可用的就是16−M−N16-M-N16−M−N条。

                            2）设一地址指令给零地址指令空余了x1x_1x1​种表示方式，二地址指令给一地址指令空余了x2x_2x2​种表示方式。

                            所以M=26⋅x1M=2^6\cdot x_1M=26⋅x1​，解出x1=M⋅2−6x_1=M\cdot2^{-6}x1​=M⋅2−6，N=x2×26−x1N=x_2\times2^6-x_1N=x2​×26−x1​，解出x2=(N+M×2−6)×2−6x_2=(N+M\times2^{-6})\times2^{-6}x2​=(N+M×2−6)×2−6。

                            同理二地址指令数量为16−x216-x_216−x2​，即24−(N+M×2−6)×2−62^4-(N+M\times2^{-6})\times2^{-6}24−(N+M×2−6)×2−6

                            最大为二地址指令只给一地址指令保留一位空余位的情况，即=16−1=15=16-1=15=16−1=15。

                            3）解题思路与第二问类似，只是变量不同，同样设一地址指令给零地址指令空余了x1x_1x1​种表示方式，二地址指令给一地址指令空余了x2x_2x2​种表示方式。

                            对于二地址指令，P=24−x2P=2^4-x_2P=24−x2​，x2=24−Px_2=2^4-Px2​=24−P。对于零地址指令，Q=26x1Q=2^6x_1Q=26x1​，x1=Q×2−6x_1=Q\times2^{-6}x1​=Q×2−6。

                            所以对于一地址指令，=x2×26−x1=(24−P)×26−Q×2−6=x_2\times2^6-x_1=(2^4-P)\times2^6-Q\times2^{-6}=x2​×26−x1​=(24−P)×26−Q×2−6。

                            例题 某机器有一个标志寄存器，其中有进位/借位标志CFCFCF、零标志ZFZFZF、符号标志SFSFSF和溢出标志OFOFOF，条件转移指令bgtbgtbgt（无符号整数比较大于时转移）的转移条件是()。

                            A.CF+OF=1A.CF+OF=1A.CF+OF=1

                            B.SF‾+ZF=1B.\overline{SF}+ZF=1B.SF+ZF=1

                            C.CF+ZF‾=1C.\overline{CF+ZF}=1C.CF+ZF​=1

                            D.CF+SF‾=1D.\overline{CF+SF}=1D.CF+SF​=1

                            解：CCC。首先问的是转移条件，无符号整数比较大于时转移，就是指两个无符号整数比较大小，假设两个无符号整数AAA和BBB，bgtbgtbgt指令会将AAA和BBB进行比较，也就是将AAA和BBB相减。若A>BA>BA>B则转移，则A−BA-BA−B肯定无进位/借位，也不为000（为000时表示两数相等），因此CFCFCF和ZFZFZF均为000，选CCC。其余选项中用到了符号标志SFSFSF和溢出标志OFOFOF，不是大于的条件，显然应当排除。

                            # 堆栈寻址

                            例题 堆栈寻址方式中，设AAA为累加器，SPSPSP为堆栈指示器，MSPM_{SP}MSP​为SPSPSP指示的栈顶单元。若进栈操作的动作是(A)→MSP(A)\rightarrow M_{SP}(A)→MSP​，(SP)−1→SP(SP)-1\rightarrow SP(SP)−1→SP，则出栈操作的动作应为()。

                            A.(MSP)→A,(SP)+1→SPA.(M_{SP})\rightarrow A,(SP)+1\rightarrow SPA.(MSP​)→A,(SP)+1→SP

                            B.(SP)+1→SP,(MSP)→AB.(SP)+1\rightarrow SP,(M_{SP})\rightarrow AB.(SP)+1→SP,(MSP​)→A

                            C.(SP)−1→SP,(MSP)→AC.(SP)-1\rightarrow SP,(M_{SP})\rightarrow AC.(SP)−1→SP,(MSP​)→A

                            D.(MSP)→A,(SP)−1→SPD.(M_{SP})\rightarrow A,(SP)-1\rightarrow SPD.(MSP​)→A,(SP)−1→SP

                            解：BBB。进、出堆栈时对栈顶指针的操作顺序是不同的，进栈时是先压入数据(A)→MSP(A)\rightarrow M_{SP}(A)→MSP​，后修改指针(SP)−1→SP(SP)-1\rightarrow SP(SP)−1→SP，说明栈指针是指向栈顶的空单元的，所以出栈时要先修改指针(SP)+1→SP(SP)+1\rightarrow SP(SP)+1→SP，然后才能弹出数据(MSP)→A(M_{SP})\rightarrow A(MSP​)→A，否则弹出的是一个空元素。

                            # 指令设计

                            某模型机共有646464种操作码，位数固定，且具有以下特点：

                            1. 采用一地址或二地址格式。
                            2. 有寄存器寻址、直接寻址和相对寻址（位移量为−128∼+127-128\sim+127−128∼+127）333种寻址方式。
                            3. 有161616个通用寄存器，算术运算和逻辑运算的操作数均在寄存器中，结果也在寄存器中。
                            4. 取数/存数指令在通用寄存器和存储器之间传送数据。

                            # 指令集计算机

                            例题 下列指令系统的特点中，有利于实现指令流水线的是()。

                            Ⅰ.指令格式规整且长度一致

                            Ⅱ.指令和数据按边界对齐存放

                            Ⅲ.只有Load/StoreLoad/StoreLoad/Store指令才能对操作数进行存储访问

                            A.A.A.仅Ⅰ、Ⅱ

                            B.B.B.仅Ⅱ、Ⅲ

                            C.C.C.仅Ⅰ、Ⅲ

                            D.D.D.I、Ⅱ、Ⅲ

                            解：DDD。指令长度一致、按边界对齐存放、仅Load/StoreLoad/StoreLoad/Store指令访存，这些都是RISCRISCRISC的特征，它们使取指令、取操作数的操作简化且时间长度固定，能够有效地简化流水线的复杂度。
