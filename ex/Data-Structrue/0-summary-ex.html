                        # 数据结构

                        例题 以下关于数据结构的说法中，正确的是()。

                        A.A.A.数据的逻辑结构独立于其存储结构

                        B.B.B.数据的存储结构独立于其逻辑结构

                        C.C.C.数据的逻辑结构唯一决定其存储结构

                        D.D.D.数据结构仅由其逻辑结构和存储结构决定

                        解：AAA。数据的逻辑结构是从面向实际问题的角度出发的，只采用抽象表达方式，独立于存储结构；而数据的存储结构是逻辑结构在计算机上的映射，它不能独立于逻辑结构而存在；数据结构包括三个要素，缺一不可。

                        # 效率度量

                        一般一个循环就是一个nnn，嵌套的两个循环就是n2n^2n2。如果实在不能计算出来就直接代入值计算。

                        首先设执行次数为ttt，然后根据基本运算（执行频率最高的语句，一般是最后一句）来计算ttt次计算后的结果，代入循环条件反推出nnn，然后来看复杂度。

                        例题 以下算法的时间复杂度为()。

                        void fun (int n) {
    int i=1;
    while(i<=n)
        i=i*2;
}

                        A.O(n)A.O(n)A.O(n)

                        B.O(n2)B.O(n^2)B.O(n2)

                        C.O(nlog⁡2n)C.O(n\log_2n)C.O(nlog2​n)

                        D.O(log⁡2n)D.O(\log_2n)D.O(log2​n)

                        解：DDD。根据基本运算：i=i*2，令执行次数为ttt，所以假如在执行中，即满足i⩽ni\leqslant ni⩽n，则ttt次时运算结果为2t2^t2t，从而2t⩽n2^t\leqslant n2t⩽n，解得t⩽log⁡2nt\leqslant\log_2nt⩽log2​n，所以时间复杂度为O(log⁡2n)O(\log_2n)O(log2​n)。

                        或者简单一点来看，默认一个循环，循环条件为一个简单的小于nnn，则时间复杂的为O(n)O(n)O(n)，而循环中对nnn进行计算，使nnn按照2n2^n2n的速度增长，所以最后时间复杂度就变成了O(log⁡2n)O(\log_2n)O(log2​n)。

                        例题 设nnn是描述问题规模的非负整数，下面的程序片段的时间复杂度是()。

                        void fun (int n) {

    x=2*x;

                        A.O(log⁡2n)A.O(\log_2n)A.O(log2​n)

                        解：AAA。基本运算为x=2*x，每执行一次就乘222，令执行次数为ttt，则第ttt运算结果为2t+1<n/22^{t+1}<n/22t+1<n/2，解得t<log⁡2(n/2)−1=log⁡2n−2t<\log_2(n/2)-1=\log_2n-2t<log2​(n/2)−1=log2​n−2，所以时间复杂度为O(log⁡2n)O(\log_2n)O(log2​n)。

                        例题 求整数nnn（n⩾0n\geqslant0n⩾0）的阶乘的算法如下，其时间复杂度是()。

                        int fact (int n) {
    if (n<=1) return 1;
    return n*fact(n-1);
}

                        A.O(log⁡2n)A.O(\log_2n)A.O(log2​n)

                        B.O(n)B.O(n)B.O(n)

                        c.O(nlog⁡2n)c.O(n\log_2n)c.O(nlog2​n)

                        D.O(n2)D.O(n^2)D.O(n2)

                        解：BBB。这显然是一个计算阶乘的程序。计算结果为n×(n−1)×⋯×1n\times(n-1)\times\cdots\times1n×(n−1)×⋯×1，一共计算nnn次，所以时间复杂度为O(n)O(n)O(n)。

                        例题 下列函数的时间复杂度是()。

                        int func(int n) {
    int i = 0, sum = 0;
    while(sum < n) sum += ++i;
    return i;
}

                        A.O(log⁡n)A.O(\log n)A.O(logn)

                        B.O(n12)B.O(n^{\frac{1}{2}})B.O(n21​)

                        C.O(n)C.O(n)C.O(n)

                        D.O(nlog⁡n)D.O(n\log n)D.O(nlogn)

                        解：BBB。首先把sum += ++i;拆开，这是两个程序语句：++i;sum += i;，即先iii自加，再sum+isum+isum+i。基本运算不能直接推出，所以使用归纳法，i=1i=1i=1，sum=0+1;sum=0+1;sum=0+1;，i=2i=2i=2，sum=0+1+2;sum=0+1+2;sum=0+1+2;，令执行次数为ttt，从而得到sum=0+1+⋯+t=(1+t)t/2sum=0+1+\cdots+t=(1+t)t/2sum=0+1+⋯+t=(1+t)t/2，所以t+t2<2nt+t^2<2nt+t2<2n，所以时间复杂度为O(n12)O(n^\frac{1}{2})O(n21​)。

                        例题 下面说法中，错误的是()。

                        Ⅰ.算法原地工作的含义是指不需要任何额外的辅助空间

                        Ⅱ.在相同规模nnn下，复杂度为O(n)O(n)O(n)的算法在时间上总是优于复杂度为O(2n)O(2^n)O(2n)的算法

                        Ⅲ.所谓时间复杂度，是指最坏情况下估算算法执行时间的一个上界

                        Ⅳ.同一个算法，实现语言的级别越高，执行效率越低

                        A.A.A.Ⅰ

                        B.B.B.Ⅰ，Ⅱ

                        C.C.C.Ⅰ，Ⅳ

                        D.D.D.Ⅰ

                        解：AAA。Ⅰ是定义，指的是耗费空间是常数级而不是不用。Ⅱ考察复杂度的定义，时间复杂度是渐进时间复杂度，不能直接赋值或利用函数图像去说必然优于。Ⅲ是对的，时间复杂度总是考虑最坏情况以保证运行时间不会比其更长。Ⅳ而言是书上所给，一般而言是对的。
